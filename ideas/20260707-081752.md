# TextForge — AI教科書オートジェネレーター

## 概要
事前にリストアップした章・項・キーワード構成をもとに、RAG強化されたLLMが教科書本文を自動生成し、LaTeX組版→PDF出力までを一気通貫で行うシステム。複数教材を並行・連続で処理できるバッチパイプライン型アーキテクチャを採用する。

## ターゲットユーザー
- 大学教員・講師（オリジナル教材を効率的に作りたい）
- 教育系スタートアップ・EdTech企業（教材量産が必要）
- 企業内研修の教材作成担当者
- 技術書・専門書の著者（初稿の叩き台を高速に作りたい）

## 解決する課題
- 教科書執筆は数ヶ月〜数年かかる重労働であり、構成が決まっていても本文化に膨大な時間がかかる
- 専門領域の正確性を担保するには参考文献・ソースの参照が必要だが、手作業での統合は非効率
- LaTeX組版の知識がないと美しいPDF教材を作れない
- 複数科目・複数バージョンの教材を同時管理・生成する仕組みがない

## 主要機能

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | 構成インポート | 章・項・キーワードのリスト（YAML/JSON/CSV）を取り込み、目次ツリーを構築 | 高 |
| 2 | RAGナレッジベース | 参考文献・PDF・Webソースをベクトルストアに格納し、生成時に検索・引用 | 高 |
| 3 | 本文生成エンジン | 章・項単位でLLMが本文を生成。キーワードを必ず含め、RAGコンテキストで正確性を補強 | 高 |
| 4 | LaTeX変換 | 生成されたMarkdown/構造化テキストをLaTeXテンプレートに変換。数式・図表・索引対応 | 高 |
| 5 | PDF出力 | LaTeXコンパイル（lualatex等）でPDFを生成。表紙・目次・ページ番号を自動付与 | 高 |
| 6 | バッチパイプライン | 複数教材定義を投入し、キュー管理で順次または並列に生成処理を実行 | 高 |
| 7 | テンプレート管理 | LaTeXテンプレート（大学講義用、研修資料用など）を複数用意し選択可能に | 中 |
| 8 | プレビュー・編集 | 生成された各章を画面上でプレビューし、人手で修正してから最終PDF化 | 中 |
| 9 | 図表自動生成 | キーワードや文脈から概念図・フローチャート・表を自動生成しLaTeXに埋め込む | 中 |
| 10 | 用語索引・相互参照 | キーワードから索引を自動生成し、章間の相互参照リンクを付与 | 中 |
| 11 | バージョン管理 | 教材ごとにリビジョンを保持し、差分確認・過去版PDF再生成が可能 | 低 |
| 12 | 多言語対応 | 同一構成から日本語版・英語版など複数言語の教材を生成 | 低 |

## 画面構成

### プロジェクト一覧画面
- 教材プロジェクトの一覧表示（名称・ステータス・最終更新日）
- 新規プロジェクト作成ボタン
- バッチ実行（選択した複数プロジェクトを一括生成）

### 構成エディタ画面
- 章・項・キーワードのツリーエディタ（ドラッグ＆ドロップで並替え）
- YAML/JSON/CSVインポート
- 各項にメモ・生成指示（トーン、深さ、対象レベル）を付与

### ナレッジベース管理画面
- 参考文献のアップロード（PDF、テキスト、URL）
- インデックス状態の確認
- ソースごとの引用プレビュー

### 生成・プレビュー画面
- 章単位で生成結果をリアルタイム表示
- インラインで本文修正可能
- LaTeXソースの直接編集タブ
- PDFプレビュー（ブラウザ内表示）

### パイプラインダッシュボード
- バッチキューの進捗状況（教材ごとの生成ステップ表示）
- 成功/失敗/再実行の管理
- 生成済みPDFのダウンロードリンク

## 技術スタック案

| レイヤー | 技術 |
|----------|------|
| バックエンド | Python (FastAPI) |
| LLM | Claude API（RAG強化） |
| ベクトルストア | ChromaDB or Qdrant |
| 埋め込みモデル | Voyage AI / OpenAI Embeddings |
| LaTeX処理 | LuaLaTeX + Jinja2テンプレート |
| PDF生成 | latexmk によるコンパイルパイプライン |
| タスクキュー | Celery + Redis（バッチ並列処理） |
| フロントエンド | バニラJS（既存プロジェクト踏襲） |
| ストレージ | ローカルファイルシステム + SQLite |

## MVPスコープ
1. **構成インポート** — YAML形式で章・項・キーワードを定義し取り込む
2. **RAGナレッジベース** — PDFをアップロードしてベクトルストアに格納
3. **本文生成** — 章・項単位でRAG付きLLMが本文を生成（1教材ずつ順次処理）
4. **LaTeX変換＋PDF出力** — 生成テキストをLaTeXテンプレートに流し込み、PDFを出力
5. **バッチ実行** — 複数の教材定義を投入して順次生成処理を実行
6. **最小UI** — プロジェクト作成 → 構成アップロード → 生成実行 → PDFダウンロードの一連のフロー
