# 実装モード クイックアクション再編

## 背景・方針

実装モードの簡易指示ボタンは9個あり、日常フローで使う頻度に差が大きい。
「開発状況に合わせてリリース段階の操作を選ぶ」という観点で、
git commit → git push → vercel deploy のパイプライン中心に再編する。

## 変更スコープ

- `client/index.html` の `#quick-actions` 内ボタンのみ
- 旧ボタンが呼んでいた `testLine()` / `testGcloud()` と対応API（`/api/test/line`・`/api/test/gcloud`）は
  他からの再利用可能性があるため**削除しない**（ボタンのみ撤去）

## 新ボタン構成（5個）

| ボタン | 動作 |
|---|---|
| 📝 commit | git add -A → 適切なメッセージでcommit |
| ⬆ push | リモートへpush（未設定なら報告） |
| 🚀 deploy | vercel本番デプロイ（未リンクならセットアップから） |
| ⚡ すべて | テスト（あれば）→ commit → push → deploy を一括実行。失敗時はそこで停止して報告 |
| 🔐 環境変数 | 秘密情報モーダルを開く（既存機能。このボタンが唯一の導線のため存置） |

### ユーザー提案からの調整点

1. **🔐 環境変数 を残す** — 秘密情報モーダル（LINE/Gmail等の認証情報管理）への唯一のUI導線のため
2. **「すべて」にテスト実行を組み込む** — デプロイ前の安全弁として、テストが存在する場合のみ実行し、
   失敗したらパイプラインを停止する

### 削除するボタン

📊 git status / ✅ テスト実行 / 🔍 構成確認 / 🔧 エラー修正 / 📱 スマホ確認 / 📤 LINEテスト / ☁ GCloud確認
（いずれも音声・テキストで指示すれば足りる）
